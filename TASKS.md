# CanvasStorm — 任务看板

## 当前产品方向

CanvasStorm 当前定位为“图形化功能发散工具”：用户从一个具体项目点开始，在功能图上继续发散可落地功能，并通过保留 / 不做完成 MVP 收敛。

## 已完成

- [x] 项目骨架：Express、ws、dotenv、JSON 文件存储
- [x] AI 代理：`/api/storm` 调用 DeepSeek，`/api/storm/status` 只返回配置状态和模型名
- [ ] CI：当前 GitHub token 缺少 `workflow` scope，workflow 暂未上传；本地 `npm test` 已覆盖
- [x] 本地登录：按用户名进入不同项目空间
- [x] 用户项目隔离：不同用户名看到不同 localStorage 项目记录
- [x] 后端用户项目空间：`/api/users/:user/projects` 按用户名读取/保存功能图项目
- [x] 前端项目同步：登录时优先读取后端项目，项目变化后同步保存，保留 localStorage 兜底
- [x] 最近项目空间：退出后登录页显示最近用户、项目数量和最近项目，可一键继续
- [x] 登录入口质感升级：右侧项目空间卡片补充按项目保存、从节点发散说明，视觉与工作台深色品牌风格统一
- [x] 新建功能图主路径：顶部和横向项目条打开创建弹窗，输入项目点和真实场景后进入自己的功能图
- [x] 图形化功能树：中心项目节点 + 功能分支 + SVG 连线
- [x] 四步收敛工作台：项目点、功能方向、下一层、MVP 取舍，用户能按一个主路径推进
- [x] 宽画布布局：项目记录改为横向切换条，主画布占据主要空间，根节点视图使用放射布局
- [x] 功能图视觉增强：画布状态条、节点类型色彩、标准 SVG viewBox 连线
- [x] 工作台降噪：项目记录横向收敛、顶部显示 4 步当前流程、右侧聚焦当前功能决策
- [x] 节点布局稳定：固定节点高度、长文案截断、默认布局升级到 v4，避免状态条和节点互相遮挡
- [x] 节点图形化：画布节点只保留名称和类型，详细说明集中到右侧，避免变成说明卡片堆
- [x] 当前路径高亮：选中功能后高亮它和上游节点，弱化无关分支
- [x] 取舍建议：按功能类型给出优先做 / 先验证 / 第二批 / 暂时不做的推荐动作
- [x] 唯一推荐焦点：选方向阶段只高亮一个建议先做节点，进入拆解/取舍后自动收起
- [x] 决策清单：右侧常驻展示 MVP 候选、先验证、暂不做和下一步验证动作
- [x] 阶段引导：画布顶部显示“现在只做什么”，生成后明确要求从当前子功能里只选一个
- [x] 当前层取舍：生成下一层后停留在父功能，右侧列出子功能选择，并在画布标记当前层“建议保留”
- [x] 视觉质感升级：深色品牌栏、深色激活项目卡、多色功能节点、柔和图形画布、绿色决策主动作
- [x] 主操作命令条：画布底部固定展示当前唯一下一步，统一承载聚焦、生成、查看建议、保留 / 不做
- [x] 聚焦渲染：项目首页只展示中心和一级方向；进入方向后隐藏无关分支，只展示当前路径和当前层子功能
- [x] 动效与交互质感：节点错峰进入、路径高亮呼吸、卡片 hover、弹窗过渡、命令条入场，并尊重 `prefers-reduced-motion`
- [x] Web Interface Guidelines 自检：表单补齐 `name` / `autocomplete`，节点移除内嵌按钮，按钮补充稳定 focus/hover/touch 反馈
- [x] AI 输出收敛：提示词和推荐排序避免评分反馈/优化机制默认成为首推，优先直观功能入口
- [x] 发散数量收敛：已有分支继续发散时默认只补充 2 个节点，避免画布一次性变乱
- [x] MiMo 参考 art direction：移除高饱和蓝色按钮和警报式红绿状态，统一为暖纸白、墨黑、鼠尾草绿、雾蓝和低饱和材料色节点
- [x] 舞台式第一屏：桌面端隐藏常驻详情栏，主界面聚焦功能图、当前动作和项目卡，减少用户第一眼的信息负担
- [x] 像素级对齐：统一页面外框、面板内距、流程块、节点和主按钮尺寸，移动端命令条不再遮挡功能图
- [x] 入口和命令条统一：登录页使用同一套舞台式视觉，桌面命令条直接显示痛点和验证依据，减少解释成本
- [x] 新建功能图弹窗：从普通表单改为同一套黑线框工具面板，明确创建后会得到中心节点、具体分支和 MVP 候选
- [x] 自定义项目首屏分支：非模板项目创建后立即带出第一层功能分支，不再让用户面对空中心节点
- [x] 场景化本地发散：AI 降级时针对菜谱、知识库、宠物健康、学习计划等项目生成具体功能，不再只给泛化功能名
- [x] 像素级收口：去掉工作区黑色转场遮挡，压淡装饰背景，移动端顶栏 / 流程条 / 项目栏改为不溢出的稳定网格
- [x] 生成等待反馈：命令条主按钮在真实 AI 慢响应时显示“生成中...”并禁用，避免用户以为没点上
- [x] 推荐路线主视觉：选方向和选子功能阶段只把唯一建议项做成主视觉，其他候选降噪为备选，减少“每个都想要”的选择负担
- [x] 移动端固定命令条：手机首屏始终能看到下一步按钮，不需要先滚动才能继续
- [x] 移动端登录入口压缩：手机首屏能直接看到用户名输入和进入按钮，减少首次进入的阅读负担
- [x] 桌面项目切换架轻量化：压缩底部项目栏高度和卡片宽度，减少表格感并放大功能图舞台
- [x] 工作台留白整理：隐藏画布统计徽标，把第一屏注意力留给推荐路线和下一步动作
- [x] 命令条降噪：隐藏首屏痛点/验证细节，只保留主建议、短说明和唯一按钮
- [x] 流程轨轻量化：四步流程从重边框卡片改为细进度轨，只突出当前步骤
- [x] 取舍完成态：保留 / 暂缓后进入“已完成”状态，命令条唯一主动作改为回到上一层
- [x] 移动端顶栏压缩：小屏只保留 CS 标记，避免品牌长字、用户名和按钮互相挤压
- [x] 移动端功能图减负：手机画布只展示当前路线和推荐节点，候选列表下放到详情区，避免首屏节点被命令条遮挡
- [x] 登录入口去工程化：入口文案改为按名字继续项目空间，隐藏次要说明块，让用户更快看到输入框和主按钮
- [x] 新建弹窗收敛：创建入口从两栏说明改为紧凑创建面板，手机首屏能直接看到创建按钮
- [x] 手机项目记录轨道：项目列表改为横向切换轨，滚到项目和详情区时隐藏固定命令条，避免遮挡内容
- [x] 完整主路径自检：桌面 / 手机走通登录、生成子功能、新建第二项目和项目切换

- [x] 键盘路径补强：登录和新建项目支持 Enter / Cmd+Enter，减少必须点按钮的操作负担

- [x] 新建弹窗主动作隔离：弹窗打开时隐藏背景命令条并锁定背景滚动，让创建路径只剩一个主动作
- [x] 登录入口文案产品化：入口卡片使用“项目记录 / 功能图空间”语言，去掉无需密码和本地演示感
- [x] 入口演示残留清理：快捷入口改为真实角色“独立开发者”，空用户名进入“我的空间”
