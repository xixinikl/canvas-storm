# CanvasStorm — AI 行为宪法

## 第一章：技术栈推荐

> 约束条件：便宜部署、初学者友好、支持 WebSocket

| 层级 | 选型 | 理由 |
|------|------|------|
| **后端运行时** | Node.js (LTS) | 前端已是纯 JS/HTML，全栈同语言，零切换成本 |
| **Web 框架** | Express.js | npm 周下载 3000 万+，中文教程最多，生态最成熟 |
| **WebSocket** | `ws` 库 | 比 Socket.IO 更轻量（18KB），直接对接浏览器原生 WebSocket API |
| **数据存储** | JSON 文件 | 会话数据就是节点树 JSON，无需 SQL；后续可平滑升级到 SQLite/PostgreSQL |
| **部署** | Railway / Render | 免费额度，Git 推上去自动部署，零运维 |
| **环境变量** | dotenv + .env | 密钥零硬编码 |

**最终推荐：Node.js + Express + ws + JSON 文件存储**

---

## 第二章：Git 死刑令

每完成 TASKS.md 里的一个 `- [ ]` 任务，**必须立即执行一次 Git 提交**。提交信息必须**精确对应任务名**，格式：

```
feat: 任务描述（与 TASKS.md 中一致）
```

例：`feat: 创建 Express 入口文件 app.js`

---

## 第三章：分支铁律

1. 所有开发在 `dev` 分支进行
2. 每开始一个新任务，从 `dev` 拉取 `feature/任务短名` 分支
3. 完成后合并回 `dev` 并推送
4. 严禁直接在 `main` 上开发
5. `main` 只接受来自 `dev` 的合并

---

## 第四章：可见性铁律

每次完成一个任务并打勾后，必须在 `CURRENT_STATUS.md` 中记录：

- 刚刚完成了什么
- 下一个要做什么

---

## 第五章：零硬编码

所有密钥（API Key、数据库密码、Token）必须存放在 `.env` 文件中，代码通过 `process.env.XXX` 读取。提供 `.env.example` 模板文件，不含真实密钥。
